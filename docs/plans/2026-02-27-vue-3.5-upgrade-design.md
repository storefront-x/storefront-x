# Design: Upgrade Vue 3.4.15 → 3.5.29

**Date**: 2026-02-27
**Branch**: vue-upgrade
**Status**: Approved

## Goal

Upgrade Vue from 3.4.15 to 3.5.29 to gain memory management improvements (56% reduction in memory usage, up to 10x faster operations on large deeply reactive arrays) and SSR fixes.

## Scope

**Single change**: `modules/vue/package.json` — update `"vue": "^3.4.15"` to `"vue": "^3.5.0"`.

After `yarn install`, `yarn.lock` resolves to Vue 3.5.29 (latest stable). Transitive dependencies (`@vue/compiler-sfc`, `@vue/server-renderer`, `@vue/reactivity`) update automatically.

No other `package.json` files need changes — Vue is declared as a dependency only in `modules/vue/package.json`. All 44 other modules get it via Yarn workspace hoisting.

## What's NOT in scope

- Vue Router upgrade (staying at 4.2.5)
- Pinia upgrade (staying at 2.1.7)
- Vite upgrade (staying at 5.4.0)
- @vitejs/plugin-vue upgrade (staying at 5.0.3)
- @vueuse/head replacement (staying at 2.0.0)

## Risk Assessment

Vue 3.5 is a minor release with no breaking changes. Edge cases:

1. **SSR hydration** — Vue 3.5 fixes stale computed values during SSR. If code relied on buggy behavior, output may change. Risk: low.
2. **Reactive props destructure** — now stable and enabled by default. If SFX doesn't use the experimental flag, nothing changes. Risk: minimal.
3. **@vueuse/* (^10.7.2)** — supports Vue 3.5 via peerDependency. Risk: none.
4. **@vue/test-utils (^2.4.4)** — supports Vue 3.5. Risk: none.

## Affected Modules

45 modules use Vue (8 with direct npm dependencies, 37 via hoisting). Largest surface area:
- `theme-tailwind` (86 files importing from `vue`)
- `theme-tailwind-magento` (32 files)
- `cms-magento` (16 files)
- `base-commerce` (14 files)

## Verification Steps

1. `yarn install` — dependencies resolve without conflicts
2. `yarn lint` — TypeScript types compatible
3. `yarn build --config storefront-x.magento.config.js` — production build passes
4. `yarn test:unit` — Vitest unit tests pass
5. `yarn test:playwright` — Playwright E2E tests pass

## Rollback

```bash
git checkout -- modules/vue/package.json yarn.lock && yarn install
```

## Vue 3.5 Key Features (for reference)

- Reactivity system overhaul: 56% memory reduction, 10x faster large reactive arrays
- Reactive props destructure (stabilized)
- `useTemplateRef()` — runtime string ID template refs
- `useId()` — SSR-stable unique IDs
- `onWatcherCleanup()` — watcher cleanup API
- Lazy hydration for async components
- `data-allow-mismatch` attribute for hydration
- Deferred Teleport
