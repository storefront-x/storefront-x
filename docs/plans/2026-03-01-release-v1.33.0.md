# Release v1.33.0 — Vue 3.5 Upgrade — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Release Storefront X v1.33.0 containing the Vue 3.4 → 3.5 upgrade to npm.

**Architecture:** Lerna fixed-mode release — `lerna version` bumps all 83 packages, generates CHANGELOG, commits, tags, and pushes. Then `lerna publish from-git` publishes to npm. GitHub Release triggers CI verification.

**Tech Stack:** Lerna v6, Yarn 3 workspaces, npm registry, GitHub Releases

---

### Task 1: Switch to main and pull latest

**Context:** The `vue-upgrade` branch has been merged into `main` on remote. We need to be on a clean, up-to-date `main`.

**Step 1: Checkout main and pull**

Run:
```bash
git checkout main && git pull
```

Expected: Branch switches to `main`, pulls merge commit(s) from `vue-upgrade`.

**Step 2: Verify clean state**

Run:
```bash
git status
```

Expected: `nothing to commit, working tree clean`

**Step 3: Verify Vue upgrade is present**

Run:
```bash
grep '"vue"' modules/vue/package.json
```

Expected: `"vue": "^3.5.0"`

**Step 4: Verify current version**

Run:
```bash
grep '"version"' lerna.json
```

Expected: `"version": "1.32.1"`

---

### Task 2: Install dependencies

**Context:** After merge, `yarn.lock` may have changed. Ensure `node_modules` are in sync.

**Step 1: Run yarn install**

Run:
```bash
yarn install
```

Expected: Resolves without errors. May show "Already up to date" or install new packages.

**Step 2: Verify no lockfile changes**

Run:
```bash
git diff yarn.lock
```

Expected: No diff (lockfile is already committed from the merge).

### Task 3: Run lint

**Context:** Verify no linting errors were introduced.

**Step 1: Run ESLint**

Run:
```bash
yarn lint
```

Expected: Exits with code 0, no errors. Warnings are acceptable.

**If it fails:** Fix linting errors before proceeding. Do NOT skip this step.

---

### Task 4: Run production build

**Context:** Verify the framework builds successfully with the Vue 3.5 upgrade. This is the most important verification step — if the build breaks, users can't use the release.

**Step 1: Build with Magento config**

Run:
```bash
yarn build --config storefront-x.magento.config.js
```

Expected: Build completes successfully. Look for "Build completed" or similar success message. No errors.

**If it fails:** This is a blocker. Debug the build error before proceeding. Common issues:
- Vue API deprecations between 3.4 and 3.5
- `@vitejs/plugin-vue` compatibility
- TypeScript type errors from Vue 3.5 changes

### Task 7: Authenticate with npm

**Context:** `lerna publish` needs npm credentials to publish `@magexo/*` packages.

**Step 1: Check npm auth**

Run:
```bash
npm whoami
```

Expected: Your npm username. If it says "Not logged in", proceed to Step 2.

**Step 2: Login to npm (only if not authenticated)**

Run:
```bash
npm login
```

Follow the interactive prompts. You need publish rights for the `@storefront-x` scope.

**Step 3: Verify access**

Run:
```bash
npm access list packages @storefront-x 2>/dev/null || echo "Check access manually on npmjs.com"
```

---

### Task 8: Lerna version bump

**Context:** This is the core release step. Lerna will:
- Update `"version": "1.32.1"` → `"1.33.0"` in `lerna.json`
- Update `"version": "1.32.1"` → `"1.33.0"` in all 83 `modules/*/package.json`
- Generate/update `CHANGELOG.md` from conventional commits
- Create git commit with message `v1.33.0`
- Create git tag `v1.33.0`
- Push commit + tag to `origin`

**Step 1: Dry run (optional but recommended)**

Run:
```bash
npx lerna version 1.33.0 --conventional-commits --no-push --no-git-tag-version --dry-run
```

Expected: Shows list of packages that will be bumped from 1.32.1 to 1.33.0. Review the list.

**Step 2: Run lerna version**

Run:
```bash
npx lerna version 1.33.0 --conventional-commits --yes
```

Expected:
- All 83 package.json files updated
- `lerna.json` updated
- `CHANGELOG.md` updated
- Git commit created
- Git tag `v1.33.0` created
- Pushed to origin

**Step 3: Verify the result**

Run:
```bash
git log --oneline -3
```

Expected: Top commit is `v1.33.0`.

Run:
```bash
git tag | grep v1.33
```

Expected: `v1.33.0`

Run:
```bash
grep '"version"' lerna.json
```

Expected: `"version": "1.33.0"`

---

### Task 9: Publish to npm

**Context:** Publishes all 83 packages to npm registry using the version from the git tag.

**Step 1: Publish**

Run:
```bash
npx lerna publish from-git --yes
```

Expected: Each package is published. Output shows `@magexo/core@1.33.0`, `@magexo/vue@1.33.0`, etc. for all packages.

**If it fails:** Common issues:
- **401/403:** npm auth expired. Run `npm login` again.
- **Package already exists:** Someone published this version already. Check npm.
- **Network error:** Retry.

**Step 2: Verify on npm**

Run:
```bash
npm view @magexo/core version
```

Expected: `1.33.0`

Run:
```bash
npm view @magexo/vue version
```

Expected: `1.33.0`

---

### Task 10: Create GitHub Release

**Context:** Creating a GitHub Release triggers the `release.yml` CI workflow, which verifies the release works by running `yarn create storefront-x` with both yarn and npm.

**Step 1: Create release via GitHub CLI**

Run:
```bash
gh release create v1.33.0 --title "v1.33.0" --notes "## Vue 3.5 Upgrade

- Bumped Vue from \`^3.4.15\` to \`^3.5.0\`
- Updated \`@vitejs/plugin-vue\` compatibility
- All 83 packages published to npm

See [CHANGELOG.md](https://github.com/storefront-x/storefront-x/blob/main/CHANGELOG.md) for details."
```

Expected: Release created. URL printed.

**Step 2: Verify CI triggered**

Run:
```bash
gh run list --workflow=release.yml --limit=1
```

Expected: Shows a run in progress or completed for `v1.33.0`.

**Step 3: Wait for CI and verify**

Run:
```bash
gh run watch
```

Expected: All jobs pass (yarn-ubuntu, create-script-with-yarn-release, create-script-with-npm-release).
