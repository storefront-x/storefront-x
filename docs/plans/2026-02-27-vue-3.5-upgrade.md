# Vue 3.5 Upgrade Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Upgrade Vue from 3.4.15 to 3.5.29 for memory management improvements (56% less memory, 10x faster large reactive arrays) and SSR fixes.

**Architecture:** Single dependency bump in `modules/vue/package.json`. Vue is the only module declaring `vue` as a direct npm dependency — all 44 other modules get it via Yarn workspace hoisting. No experimental flags or deprecated APIs are in use.

**Tech Stack:** Vue 3.5, Yarn 3 workspaces, Vitest, Playwright, Vite 5.4.0

**Design doc:** `docs/plans/2026-02-27-vue-3.5-upgrade-design.md`

### Task 3: Run lint to check TypeScript compatibility

Vue 3.5 ships updated type definitions. This step catches any type incompatibilities across all 45 modules that use Vue.

**Step 1: Run ESLint**

Run: `yarn lint`
Expected: PASS with no new errors.

**If lint fails:** The most likely cause is changed type signatures in Vue 3.5's reactivity system. Fix the type errors before proceeding. Common changes in 3.5 types:
- `ComponentPublicInstance` type refinements
- `DefineComponent` generic parameters
- `useTemplateRef` vs `ref` for template refs (additive, not breaking)

**Step 2: Commit any fixes (if needed)**

```bash
git add -A
git commit -m "fix: resolve type errors after vue 3.5 upgrade"
```

This verifies that Vite + @vitejs/plugin-vue 5.0.3 can compile all `.vue` SFCs and produce a working production bundle with Vue 3.5.

**Step 1: Run build**

Run: `yarn build --config storefront-x.magento.config.js`
Expected: PASS — build completes without errors.

**Key things the build validates:**
- `@vitejs/plugin-vue@5.0.3` compatibility with Vue 3.5 (should be fine — plugin-vue 5.x supports Vue 3.4+)

**If build fails:** Check the error message. Most likely causes:

- `vue-demi` alias in `modules/vue/vite.config.js` — the alias `'vue-demi': 'vue-demi/lib/v3/index.mjs'` should still work, but verify if the error points there

### Task 5: Run unit tests

**Step 1: Run Vitest**

Run: `yarn test:unit`
Expected: PASS — all unit tests pass.

**Context:** Unit tests run in jsdom via `vitest.config.js`. They use `@vue/test-utils@^2.4.4` which supports Vue 3.5. Tests are sourced from `.sfx/tests/unit/` (concept-generated from modules).

**If tests fail:** The most likely cause is changed reactivity timing in Vue 3.5. The reactivity system was rewritten for memory efficiency, which could affect:
- `nextTick` timing in tests
- Computed value evaluation order
- Watcher trigger timing

Fix failing tests case-by-case. These would be genuine behavior changes, not regressions.

---

### Task 6: Run Playwright E2E tests

**Step 1: Run Playwright**

Run: `yarn test:playwright`
Expected: PASS — all E2E tests pass (with up to 2 retries per test, as configured in `playwright.config.js`).

**Context:** Playwright tests live in `modules/**/tests/playwright/*.spec.js`. They have global setup/teardown in `tests/playwright/`. These tests exercise the full SSR + hydration pipeline, which is the area most impacted by Vue 3.5's reactivity changes.

**If tests fail:** Check whether failures are:
- **Hydration mismatches** — Vue 3.5 fixes stale computed values during SSR, which could change rendered output. If the fix is correct, update the test expectation.
- **Timing issues** — not `waitForTimeout` (anti-pattern), but legitimate race conditions exposed by faster reactivity.
- **Flaky tests** — retries are set to 2. If a test passes on retry, it was likely flaky before.

**Step 2: Commit any test fixes (if needed)**

```bash
git add -A
git commit -m "fix: update tests for vue 3.5 compatibility"
```

## Rollback Procedure

If at any point the upgrade causes unresolvable issues:

```bash
git checkout -- modules/vue/package.json yarn.lock && yarn install
```

This restores Vue 3.4.15 immediately. All changes are on the `vue-upgrade` branch, so `main` is never affected.
